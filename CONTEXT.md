# Hanzi Studio

Hanzi Studio is a visual editor for Hanzi SVG character artwork. The product exists to let a user select a character SVG and experiment with shader-based visual effects, not to publish, export, mint, store, or generate assets.

## Language

**Hanzi Studio**:
The product context for experimenting with shader effects on Hanzi SVG character shapes.
_Avoid_: minting app, NFT studio, export tool

**Introduction Page**:
The entry page that introduces Hanzi Studio before the user enters the editor.
_Avoid_: editor, mint entry

**Character Editor**:
The workspace where a user selects a Hanzi character SVG and adjusts shader effect settings.
_Avoid_: mint page, upload flow, NFT creator, text editor

**Shader Effect View**:
The live preview where the selected Hanzi character SVG is rendered as a character mesh with the selected shader effect.
_Avoid_: export preview, mint preview, upload preview, SVG filter preview, flat mask preview

**Character Mesh**:
A renderable extruded geometry derived from the selected Hanzi character SVG path.
_Avoid_: text mesh, font glyph, image mask

**Displacement Map**:
A pattern image, either built in or locally uploaded, used to distort the **Character Mesh** or shader sampling.
_Avoid_: background image, AI background, DALL-E image

**Character Panel**:
The editor control panel for selecting the Hanzi character SVG.
_Avoid_: Text panel, typography panel

**Shader Effect Preset**:
A built-in shader option that defines an effect, its user-adjustable parameters, and default values.
_Avoid_: background image, AI background, DALL-E image

## Relationships

- **Hanzi Studio** has one **Introduction Page** and one primary **Character Editor**.
- The **Introduction Page** leads into the **Character Editor**.
- A **Character Editor** includes a **Character Panel** for selecting the Hanzi character SVG.
- A **Character Editor** produces one live **Shader Effect View**.
- A **Character Editor** derives one **Character Mesh** from the selected character SVG.
- A **Character Editor** applies one selected **Shader Effect Preset** to the **Character Mesh**.
- A **Character Mesh** has global mesh controls for extrusion depth, character mesh thickness, rotation, scale, position, and auto-rotation.
- A **Character Mesh** may use one selected **Displacement Map** for geometry displacement.
- A **Shader Effect View** may use the same **Displacement Map** for image-based shader distortion.
- **Shader Effect View** is local to the browser session and is not exported, stored remotely, minted, or generated by an AI service.
- **Character Editor** settings persist locally so a page refresh does not accidentally discard work.
- The **Character Editor** exposes controls only; it has no queue, mint, export, remote upload, or modal action workflow.

## Example dialogue

> **Dev:** "After the user adjusts the shader on a character, should we offer export or mint actions?"
> **Domain expert:** "No. **Hanzi Studio** is only for experimenting with shader effects in the **Character Editor**."

## Flagged ambiguities

- "Next.js 16 app architecture" does not mean a product expansion. The repo already declares Next.js 16 and uses `app/`; the architectural goal is to align the app around **Hanzi Studio** and remove legacy Web3, NFT, upload, queue, export, and AI-generation flows.
- "SVG effect playground" is retired as the current product target. The **Character Editor** now uses a **Shader Effect View** backed by a **Character Mesh** while preserving character SVG selection.
- "Extrude level" should be called extrusion depth in user-facing and planning language.
- Character mesh thickness is separate from extrusion depth. Character mesh thickness changes the planar weight of the character shape; extrusion depth changes the z-depth of the extruded mesh.
- Mesh controls are separate from shader preset parameters. Mesh controls change geometry, object transform, or animation; shader preset parameters change uniforms for the selected shader.
- Displacement map controls are global editor controls, not shader preset parameters. The selected pattern image can drive both mesh displacement and image-based shader distortion for easier experimentation. Local Displacement Map uploads are session-only inputs, not remote upload or asset-storage workflows.
- "Metadata" referred to NFT metadata fields and is removed from the editor. Character identity may still exist only as part of SVG character selection.
- Contract and chain tooling is not part of **Hanzi Studio**. The full `hardhat/` workspace and contract-related state belong to the removed minting product boundary.
- Public product copy should describe Hanzi SVG character shader editing, not NFT, minting, wallet, chain, or Optimism positioning.
- Studio-specific UI should use Studio language: **Character Panel**, shader presets, shader parameters, and **Shader Effect View**.
- Global app chrome should not own Studio editor state; Studio state belongs to the **Character Editor** route boundary.
- Studio settings are locally persisted for refresh safety, but they are not synced to a backend or exported as files in v1.
