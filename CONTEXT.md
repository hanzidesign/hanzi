# Hanzi Studio

Hanzi Studio is a visual editor for Hanzi SVG character artwork. The product exists to let a user inspect and adjust SVG-based character effects, not to publish, export, mint, upload, or generate assets.

## Language

**Hanzi Studio**:
The product context for editing and viewing Hanzi SVG character effects.
_Avoid_: minting app, NFT studio, export tool

**Introduction Page**:
The entry page that introduces Hanzi Studio before the user enters the editor.
_Avoid_: editor, mint entry

**Character Editor**:
The workspace where a user selects a Hanzi character SVG and adjusts its visual effect settings.
_Avoid_: mint page, upload flow, NFT creator, text editor

**SVG Effect View**:
The rendered view of the selected Hanzi character SVG with the current effect and style settings applied.
_Avoid_: export preview, mint preview, upload preview

**Character Panel**:
The editor control panel for selecting the Hanzi character SVG.
_Avoid_: Text panel, typography panel

**SVG Effect Pattern**:
A local pattern used as part of the SVG effect system in the character editor.
_Avoid_: background image, AI background, DALL-E image

## Relationships

- **Hanzi Studio** has one **Introduction Page** and one primary **Character Editor**.
- The **Introduction Page** leads into the **Character Editor**.
- A **Character Editor** includes a **Character Panel** for selecting the Hanzi character SVG.
- A **Character Editor** produces one live **SVG Effect View**.
- A **Character Editor** may adjust an **SVG Effect Pattern** as part of the **SVG Effect View**.
- **SVG Effect View** is local to the browser session and is not exported, uploaded, minted, or generated by an AI service.
- The **Character Editor** exposes controls only; it has no queue, mint, upload, export, or modal action workflow.

## Example dialogue

> **Dev:** "After the user adjusts the character effect, should we offer export or mint actions?"
> **Domain expert:** "No. **Hanzi Studio** is only for viewing SVG character effects in the **Character Editor**."

## Flagged ambiguities

- "Next.js 16 app architecture" does not mean a product expansion. The repo already declares Next.js 16 and uses `app/`; the architectural goal is to align the app around **Hanzi Studio** and remove legacy Web3, NFT, upload, queue, export, and AI-generation flows.
- "Background" referred to AI/image background tooling and is removed. **SVG Effect Pattern** is distinct from background and remains because pattern is one of the SVG effects.
- "Metadata" referred to NFT metadata fields and is removed from the editor. Character identity may still exist only as part of SVG character selection.
- Contract and chain tooling is not part of **Hanzi Studio**. The full `hardhat/` workspace and contract-related state belong to the removed minting product boundary.
- Public product copy should describe Hanzi SVG character effect editing, not NFT, minting, wallet, chain, or Optimism positioning.
- Studio-specific UI should use Studio language: **Character Panel**, effect controls, style controls, and **SVG Effect View**.
- Global app chrome should not own Studio editor state; Studio state belongs to the **Character Editor** route boundary.
- Studio settings are session-local unless a future product decision explicitly adds persistence.
